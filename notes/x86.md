## Registros
Tiene varios registros hechos en especifico para ciertas cosas.
	Es asi porque el 8066 era muy limitado y se hizo asi para sacarle el mejor provecho. Se mantuvo asi por backwards compatibility

- General registers
	- Indices registers: punteros e indices
	- Data registers:
		- AX: para arimetrica e i/o general
- Status registers
	- Program counter: En que ubicacion de memoria empieza la instruccion actual
	- Flags register: Flags especiales
- Segment registers: Trackean los segmentos de memoria activos
	- CS: el segmento activo del cual se esta ejecutando codigo
		- IP / program counter: el offset
	- ES, FS, GS: Segmentos de datos extra
	- SS: el stack register
	- DS: data segment, el que uso para yo guardar cosas
		- SI: el offset
## Como acceder
Como no puedo acceder a un a posicion de memoria > 16 bits(no me da el registro), tengo que referenciar direcciones de esta manera(todos los campos son opcionales)

`<register>: [base + index * scale + displacement]`
- register: CS, DS, ES, FS, GS, SS(DS si no se especifica)
- base: BP/BX para 16 bits, cualquier registro general para 32/64
- index: SI/DI para 16 bits, cualquier general para 32/64
- scale: solo para 32/64 bits. 1, 2, 4, 8
- displacement: una constante(con signo)

## Stack
Cuando se llama una funcion, la direccion de la siguiente linea a ejecutar se guarda en el stack. **Si mi funcion toca el stack y no lo vuelve a dejar como estaba antes, el programa crashea**.
